The one mental model

Copilot usage-based billing combines fixed monthly seat fees with shared AI credit pools.

ConceptDefinition & Impact
Seat / LicenseFixed monthly cost ($19 Business / $39 Enterprise). Required for access.
Included CreditsMonthly AI credit allowance granted by each active seat.
The Shared PoolAggregation of all seat credits into a single pot for the entire enterprise or organization.
Metered UsageOverage charges at $0.01 / AI credit once the shared pool is exhausted.
Base FeaturesCode completions + next-edit suggestions are included in seat fees and do not consume AI credits.
Governing BudgetsSpending caps at user, cost center, organization, or enterprise level to control metered risk.
Billing CyclePool resets 00:00 UTC, 1st of the month (credits do not roll over).
The core principle: You pay a fixed seat fee to fund a shared credit bucket. Requests are deducted from the bucket until empty, then billed as metered overage unless a budget stops it.

Why the same task costs different amounts

Copilot cost changes based on how much work the AI has to do.

What changes cost?Plain-English meaning
Model selectedMore powerful models cost more. Use them for harder work, not every task.
Amount of contextAsking Copilot to read more files, history, or pasted text uses more credits.
Size of the answerLonger generated answers cost more than short answers.
Simple rule: Bigger model + more files + longer answer = higher cost. Keep routine work small and use powerful models only when they add value.

Budget controls

Two families. Learn the split and the rest follows.

BudgetCaps whatActive whenHard stop?
Universal user-levelEach user's total credit useAlways (pool + metered)Always
Cost-center user-levelEach member's total use (per cost center)AlwaysAlways
Individual user-levelOne specific user's total useAlwaysAlways
Cost center budgetA team's metered chargesMetered phase onlyOnly if "Stop usage" is enabled
Organization budgetAn organization's metered chargesMetered phase onlyOnly if "Stop usage" is enabled
Enterprise budgetAll metered charges (failsafe)Metered phase onlyOnly if "Stop usage" is enabled

User-level budgets

Controls how much each person can use in total.

Applies from the first credit used, whether credits come from the included pool or paid overage. When the person reaches the limit, usage stops.

Group budgets (cost center / organization / enterprise)

Controls how much paid overage a team, organization, or enterprise can spend.

Only matters after included credits are used up. It blocks usage only when “Stop usage” is enabled.

Important default: “Stop usage when budget limit is reached” is off by default on group budgets. If this is not enabled, charges can continue after the budget threshold is reached.

Precedence rules

  • Most specific user-level wins: Individual → Cost-center user-level → Universal.
  • Lowest remaining headroom wins — even the enterprise budget can block a user who still has personal room.
  • Any $0 budget = immediate block for whoever it covers.
  • Raising a group budget does not unblock someone who hit their user-level budget (and vice versa).

Incident runbook — users blocked or credits exhausted

Goal: restore critical developer work without opening unlimited spend.

Golden rule: Do not remove caps or turn off hard stops to resolve the incident quickly. First identify what limit was hit, then unblock only the users or teams that need it.

1. Confirm the blocker

Find out what actually stopped usage.

  • Pool empty: included credits are used up.
  • User limit hit: one person reached their cap.
  • Group cap hit: team, organization, or enterprise overage cap stopped usage.

2. Communicate immediately

Reduce panic while the fix is being prepared.

  • Tell developers: code completions and next-edit suggestions still work.
  • Tell managers: max exposure is controlled by the overage cap.
  • Share an ETA for the first usage readout.

3. Find the usage driver

Identify whether this is normal demand or a spike.

  • Start with AI usage: set the Time Frame, then use Filter and Group to find top consumers.
  • Find the pattern: check whether spend is concentrated in a few users, one organization, one cost center, or one model.
  • Export evidence when needed: use Get usage report for date, user, and model detail.
  • Check burn rate: compare credits used with days elapsed in the billing month. Early-cycle exhaustion signals a usage spike.
How to check this in practice

Where to start

Start in GitHub billing and licensing views. Use AI usage for usage patterns, then check licensing and organization access for seat context.

  1. Open AI usage: go to Billing and licensing → Usage → AI usage, then review Copilot usage across licensed users, teams, organizations, or cost centers. See image 1.
  2. Check licensing and access: in the enterprise, go to Billing and licensing → Licensing → Copilot → Manage to confirm license counts and billing context. See image 2A. For user-level access, open the organization's Settings → Copilot → Access page. See image 2B.
  3. Find the top consumers: on Billing and licensing → Usage → AI usage, set the Time Frame, then use Filter and Group to identify the highest-consuming users, organizations, cost centers, and models.
  4. Export the evidence: from AI usage, click Get usage report to review AI credits by date, user, and model. Use Metered usage → Get usage report only when product/SKU-level billing detail is needed.
  5. Check model mix: in AI usage, group or report by model to find which models drive the most credits. Compare high-cost models against the Models and pricing reference, then flag routine high-cost usage for engineering review.
  6. Check timing: on AI usage, set Time Frame to the current billing month and compare credits used with days elapsed. Early-month exhaustion signals a spike; near month-end usage may be normal demand. Use Get usage report for UTC date-level evidence.
  7. Decide action: increase a budget only when the usage is expected and business-critical. Use Billing and licensing → Budgets and alerts to update the specific user, team, organization, or enterprise budget that was hit. If the usage looks unnecessary, ask engineering or Copilot policy owners to review it first.

4. Unblock safely

Restore access without creating a blank check.

  • Raise limits only for approved power users or critical teams.
  • Keep enterprise and group caps in place with hard stops enabled.
  • Re-check sizing math before increasing broad limits.

$10 budget= 1,000 AI credits

Included AI credits (Enterprise)= seats × 3,900 credits
When overage can happenWhen approved user limits are higher than the included credits available.
Extra budget to planSet an overage budget for usage that goes beyond the included credits.
Highest planned monthly bill= license fees + approved overage cap
Before using this for billing decisions: Prices and included AI credit allowances can change. Validate against current GitHub Docs and your GitHub agreement.
Common billing interpretation mistake: 400 Business seats × $19 = $7,600 fixed. A $5,000 enterprise budget does not mean a $5,000 total bill — it means highest planned bill = $7,600 + $5,000 = $12,600. The enterprise budget caps extra usage only.
